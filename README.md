## 📄 Paper Summary

### Abstract

**AutoVLA: A Vision-Language-Action Model for End-to-End Autonomous Driving with Adaptive Reasoning and Reinforcement Fine-Tuning**

AutoVLA is an end-to-end autonomous driving framework that unifies semantic reasoning and trajectory planning within a single autoregressive generative model. The key innovations are:

1. **Unified Architecture**: Integrates a pretrained Vision-Language Model (VLM) with physical action tokens, enabling direct trajectory prediction without intermediate representations.

2. **Dual Thinking Modes**:
   - **Fast Thinking**: Direct trajectory generation without reasoning (for simple scenarios)
   - **Slow Thinking**: Chain-of-Thought (CoT) reasoning + trajectory (for complex scenarios)

3. **Two-Stage Training**:
   - **SFT (Supervised Fine-Tuning)**: Trains on both trajectory-only and CoT-augmented data using Qwen2.5-VL-3B backbone
   - **RFT (Reinforcement Fine-Tuning)**: Uses Group Relative Policy Optimization (GRPO) with task-specific reward functions to improve performance and reduce unnecessary reasoning

4. **Action Tokenization**: Continuous trajectories discretized into K=2048 discrete tokens via K-disk clustering, enabling autoregressive planning within language models.

**Backbone Model**: Qwen2.5-VL-3B (3 billion parameters, open-source, deployment-friendly)

**Datasets Used**: nuPlan, nuScenes, Waymo, CARLA

**Results**: Competitive performance across multiple benchmarks with adaptive fast/slow thinking capabilities.

---

### Introduction

#### Problem Statement

Current end-to-end autonomous driving approaches face two critical limitations:

1. **Physically Infeasible Action Generation**:
   - Many VLM-based methods generate text waypoints or actions that violate physical constraints
   - Intermediate meta-actions or latent tokens require downstream processing, breaking end-to-end optimization
   - Mode collapse is common in trajectory generation

2. **Inflexible Reasoning Strategy**:
   - Most models use fixed reasoning (always CoT or never CoT)
   - Cannot adaptively switch between direct output (simple cases) and reasoning (complex cases)
   - Results in unnecessary computation and runtime overhead

#### AutoVLA's Solution

AutoVLA addresses these problems through three key contributions:

1. **Integrated Physical Action Tokens**:
   - Discretizes continuous trajectories into K=2048 physically feasible tokens
   - Tokens represent 0.5-second vehicle motions (Δx, Δy, Δθ)
   - Directly integrated into VLM vocabulary, enabling end-to-end training

2. **Adaptive Dual-Process Reasoning**:
   - Unified single-model architecture with two thinking modes
   - Model learns when to use fast vs slow thinking during SFT
   - GRPO-based RFT penalizes unnecessary reasoning while improving planning

3. **Scalable Training Pipeline**:
   - Automated CoT annotation pipeline using Qwen2.5-VL-72B
   - Mixed training on multiple datasets (nuPlan: 166.3k, nuScenes: 19.0k, Waymo: 23.8k)
   - Achieves 10.6% PDMS improvement with 66.8% runtime reduction after RFT

#### Key Advantages Over Baselines

| Feature | AutoVLA | Other VLMs | Other E2E Methods |
|---------|---------|-----------|------------------|
| **End-to-End Unified** | ✅ Yes | ⚠️ Partial | ✅ Yes |
| **Physical Action Tokens** | ✅ Yes | ❌ No | ⚠️ Simple |
| **Adaptive Reasoning** | ✅ Fast+Slow | ❌ Fixed | ❌ No reasoning |
| **World Knowledge** | ✅ VLM-based | ✅ VLM-based | ❌ Limited |
| **Model Complexity** | ✅ Simple | ❌ Complex | ✅ Simple |
| **Real-time Capable** | ✅ 1 Hz | ❌ Slow | ✅ 2 Hz |

## 🎯 NuScenes-Only Reproduction Plan

This reproduction focuses solely on the **nuScenes dataset** to demonstrate paper's key concepts.

### Why NuScenes Only?

#### 📊 Data Size Comparison

| Dataset | Size | Training Time | GPUs | Purpose in Paper |
|---------|------|---|---|---|
| **nuPlan** | 166.3k samples (~500 GB) | 48+ hours | 8x L40S | Closed-loop benchmark |
| **nuScenes** | 19.0k samples (~302-354 GB) | ~30h (SFT) + 197.4h (RFT) | 1x V100S | Open-loop benchmark ✅ |
| **Waymo** | 23.8k samples (~200 GB) | 24-36 hours | 8x L40S | Long-tail scenarios |
| **CARLA** | 274.5k samples (~100 GB) | 36+ hours | 4-8x GPUs | Closed-loop sim |

**Our Choice: NuScenes**
- ✅ **Smallest dataset** (~302-354 GB compressed/uncompressed) - manageable compared to nuPlan (500GB)
- ✅ **Fastest to process** (~30h SFT + 197.4h RFT = 227h total ≈ 9.5 days, vs 48h+ for nuPlan)
- ✅ **Demonstrates core concepts** - enough samples for meaningful results
- ✅ **Urban focus** - unique scenario diversity (Boston, Singapore)
- ✅ **Open-loop evaluation** - standard metrics well-defined
- ❌ Does NOT require CARLA simulator
- ❌ Does NOT require nuPlan's complex routing system

**What We Trade-Off**:
- Paper trains on ALL 4 datasets → better generalization
- We train on NuScenes only → focused results, faster iteration
- Paper reports NAVSIM (nuPlan) → we report nuScenes metrics
- **Result**: Demonstrates AutoVLA's adaptability on single dataset

#### Table 2: Runtime Analysis - Fast vs Slow Thinking
**Dataset**: 500 random NuScenes validation samples

**What We Measure**:
- **Fast Thinking**: Direct action generation without CoT reasoning
  - No reasoning tokens generated
  - Just trajectory tokens (10 tokens × 0.5s = 5s horizon)
  - Paper: Min=0.997s, Max=1.116s, **Avg=1.072s**

- **Slow Thinking**: Full chain-of-thought + action generation
  - Generate ~500-1000 reasoning tokens first
  - Then generate trajectory tokens
  - Paper: Min=7.607s, Max=13.706s, **Avg=10.518s**

**Why Slow is 9.8x Slower?**
```
Fast:  input → (tokenization) → output = ~1.0s
Slow:  input → (reasoning: 500 tokens) → (tokenization: 10 tokens) → output = ~10.5s
       └─── Autoregressive generation is sequential ────────────────┘
```

**RFT Benefit**: After RFT with CoT length penalty, model learns to use fast thinking 
in simple scenarios → reduces average from 10.5s to ~0.7s (67% reduction)

**Log Location**: `logs/12_evaluate.out` (timing subsection)

## 📊 Metrics Definitions

### L2 Distance (Average Displacement Error)
```
L2_error = √( (x_pred - x_gt)² + (y_pred - y_gt)² )
Computed at each timestep, averaged across trajectory
Units: meters
```

### Collision Rate
```
collision_rate = (# predictions colliding with objects) / (# total predictions)
Measured using semantic segmentation maps
Units: percentage
```

### PDMS (Predictive Driver Model Score)
- Combines: No-at-fault collision, drivable area compliance, ego progress, comfort, TTC
- Range: [0-100], higher is better

### Runtime (our V100 measurement, 100 samples)
```
fast_thinking: min=0.93s, max=9.54s, mean=4.62s
slow_thinking: min=1.43s, max=3.43s, mean=2.93s
```
Note: on V100 with max_new_tokens=64 the fast/slow ratio differs from the paper (paper: A100, full CoT).
Paper reports fast=1.07s, slow=10.52s (9.8× slower).
